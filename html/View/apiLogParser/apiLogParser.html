使用說明｜API Log 解析工具

1️⃣ 將 原始 Log 貼到上方「原始日誌」輸入框。
支援自動排除Linux查詢指令，系統只會抓出 JSON 區段解析。

2️⃣ 解析成功後會自動產生卡片列表：
・綠色 OK：通常代表狀態正常
・紅色 ERROR：偵測到錯誤狀態（例如 HTTP status ≥ 400、retCode=0、或有 errCode）

3️⃣ 操作方式：
・點卡片標題列可 展開/收合 Request / Response
・可用上方搜尋框搜尋：API 名稱、路由、錯誤碼等內容
・展開全部 Error：只展開錯誤卡片
・全部展開：一次展開所有卡片

4️⃣ 右側顯示的檔名來源
範例：apilog_2025-10-09-13:00:00.log... {JSON}
系統會把 JSON 前面的內容當成來源，顯示在卡片右側。

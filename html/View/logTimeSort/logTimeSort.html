MG LOG 排序

1. 輸入 LOG 📝
將 MG / WebAPI LOG 貼進來，會自動依日期時間欄位排序。
小提示：Ctrl + Enter 也可開始解析。
2. 原始 Log 排序結果
依時間由舊到新排序後，保留原始整行內容。
3. 選擇顯示欄位 ⚙️
點選下方分類展開選單，勾選欲顯示的欄位。
[表格欄位]可[拖曳]標題重新排序欄位順序。
4. 排序結果表格（由舊到新）⬇️
每一列代表一行 LOG，會展開 JSON data 欄位，讓你用表格方式檢視。

使用說明

1️⃣ 將 MG / WebAPI / LOG 貼到第 1 步「輸入 LOG」區塊
需包含 ,data=，例如：
2025/12/19 14:36:15 ... /api/xxx?x=1 ,data={"update":"{...}"}

2️⃣ 點「解析並排序 LOG 」或按 Ctrl + Enter 進行解析。
系統會依 Time 欄位由舊到新排序，並同時輸出「原始 Log 排序結果」。

3️⃣ 「選擇顯示欄位」可以：
・展開分類勾選要顯示的欄位
・使用「搜尋欄位」快速找英文 key 或中文名稱
・按「更新列表 🔄」把表格欄位更新成你目前勾選的欄位

4️⃣ 表格操作：
・每一列代表一行 log（會展開 data 內的欄位）
・可拖曳表格標題（欄位名）調整欄位順序
・若 log 內含 bike_gps，會自動計算 GPS_Decimal（十進位緯經度）

5️⃣ 常見解析失敗原因：
・整行沒有 ,data=
・data= 後面不是合法 JSON（缺括號、引號被截斷）
・多行貼上時，某些行不是 log 內容（可先刪掉無關行）
